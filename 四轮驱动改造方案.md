# 四轮驱动改造方案

## 问题分析

### 1. 当前架构问题
- 系统使用"板"概念，每个板有turn_node和drive_node两个节点
- 四驱车需要4个独立轮子节点：前左、前右、后左、后右
- 当前架构不适合四轮独立控制

### 2. 闭环后移动问题
从日志分析：
```
[16:38:30] CANDLE TX Set_Input_Vel velocity=2.0
[16:38:30] CANDLE TX Set_Input_Vel velocity=0.0  // 立即发送停止命令
```
问题：系统在发送前进命令后立即发送停止命令，导致电机只动一下就停

### 3. 高级界面节点选择灰色问题
- 节点下拉框可能未正确启用
- 需要在节点检测后启用选择功能

## 解决方案

### 方案A：扩展现有架构（推荐）
将每个"板"的两个节点扩展为4个轮子节点：
- frontLeftNode (前左轮)
- frontRightNode (前右轮)  
- rearLeftNode (后左轮)
- rearRightNode (后右轮)

优点：
- 改动较小，兼容现有代码
- 可以复用现有的板管理逻辑
- 配置文件结构变化小

### 方案B：重构为轮子架构
完全重构为4个独立轮子的架构

优点：
- 架构更清晰
- 更符合四驱车的概念模型

缺点：
- 改动大，风险高

## 推荐实施方案A

### 修改内容

#### 1. 数据结构修改
```cpp
struct BoardConfigRow {
    // 原有的 turnNodeCombo, driveNodeCombo 改为：
    QComboBox *frontLeftNodeCombo;
    QComboBox *frontRightNodeCombo;
    QComboBox *rearLeftNodeCombo;
    QComboBox *rearRightNodeCombo;
    
    QDoubleSpinBox *frontLeftScaleSpin;
    QDoubleSpinBox *frontRightScaleSpin;
    QDoubleSpinBox *rearLeftScaleSpin;
    QDoubleSpinBox *rearRightScaleSpin;
    
    int preferredFrontLeftNodeId;
    int preferredFrontRightNodeId;
    int preferredRearLeftNodeId;
    int preferredRearRightNodeId;
};
```

#### 2. 运动控制逻辑
- 前进：4个轮子同时发送相同速度命令
- 后退：4个轮子同时发送相反速度命令
- 左转：左侧轮子减速/反转，右侧轮子加速
- 右转：右侧轮子减速/反转，左侧轮子加速

#### 3. 同步控制
- 使用批量命令发送，减少延迟
- 添加同步状态监控，显示4个轮子的实时状态
- 添加速度差异检测和补偿

#### 4. 修复闭环移动问题
关键修改：
```cpp
void MainWindow::startManualMotion(bool turnAxis, double direction, QPushButton *sourceButton)
{
    // 保存运动状态，不要立即发送停止命令
    m_activeManualButton = sourceButton;
    m_manualMotionDirection = direction;
    
    // 发送速度命令到所有轮子
    // 不要在这里发送停止命令
}

void MainWindow::stopManualMotion()
{
    // 只在这里发送停止命令
    m_activeManualButton = nullptr;
    // 发送0速度到所有轮子
}
```

#### 5. 启用节点选择
```cpp
void MainWindow::updateBoardNodeCombos(int boardIndex)
{
    // 填充节点列表后
    combo->setEnabled(true);  // 确保启用
}
```
