# 在高级面板添加速度配置监控显示方案

## 需求

在 Qt 程序的高级面板状态监控中添加以下信息的实时显示：
- 速度限制 (vel_limit)
- 加速度限制 (vel_ramp_rate)
- 位置增益 (pos_gain)

这样用户可以实时看到当前的速度配置，避免速度设置过高或过低的问题。

## 实现方案

### 1. 在 mainwindow.h 中添加新的标签成员变量

在 `MainWindow` 类的 private 部分添加：

```cpp
// 在现有的状态标签后面添加
QLabel *m_velLimitValue;        // 速度限制
QLabel *m_velRampRateValue;     // 加速度限制
QLabel *m_posGainValue;         // 位置增益
```

### 2. 在 buildAdvancedContent 函数中创建显示控件

在状态监控部分（statusLayout）添加新的行：

```cpp
// 在现有的状态显示后面添加（例如在 m_temperatureValue 之后）

statusLayout->addWidget(new QLabel("速度限制"), row, 0);
m_velLimitValue = createValueLabel();
statusLayout->addWidget(m_velLimitValue, row++, 1);

statusLayout->addWidget(new QLabel("加速度限制"), row, 0);
m_velRampRateValue = createValueLabel();
statusLayout->addWidget(m_velRampRateValue, row++, 1);

statusLayout->addWidget(new QLabel("位置增益"), row, 0);
m_posGainValue = createValueLabel();
statusLayout->addWidget(m_posGainValue, row++, 1);
```

### 3. 在 MainWindow 构造函数中初始化

```cpp
m_velLimitValue(nullptr),
m_velRampRateValue(nullptr),
m_posGainValue(nullptr),
```

### 4. 修改 ODriveMotorController 以获取配置信息

需要在 `ODriveMotorController` 类中添加获取配置信息的功能。

#### 4.1 在 odrivemotorcontroller.h 的 AxisStatus 结构中添加：

```cpp
struct AxisStatus
{
    // ... 现有字段 ...
    
    // 添加配置信息
    float velLimit = 0.0f;
    float velRampRate = 0.0f;
    float posGain = 0.0f;
};
```

#### 4.2 在 odrivemotorcontroller.cpp 中解析 Get_Controller_Config 响应

需要发送 `Get_Controller_Config` (0x014) 命令并解析响应。

### 5. 在 updateStatusPanel 函数中更新显示

```cpp
void MainWindow::updateStatusPanel()
{
    // ... 现有代码 ...
    
    // 添加速度配置显示
    if (m_velLimitValue) {
        if (status.velLimit > 0.0f) {
            m_velLimitValue->setText(QString("%1 转/秒").arg(QString::number(status.velLimit, 'f', 2)));
            
            // 根据速度值设置颜色警告
            if (status.velLimit > 3.0f) {
                m_velLimitValue->setStyleSheet("QLabel { color: #d32f2f; font-weight: 600; }"); // 红色：过高
            } else if (status.velLimit > 2.0f) {
                m_velLimitValue->setStyleSheet("QLabel { color: #f57c00; font-weight: 600; }"); // 橙色：较高
            } else if (status.velLimit >= 0.5f) {
                m_velLimitValue->setStyleSheet("QLabel { color: #388e3c; font-weight: 600; }"); // 绿色：合理
            } else {
                m_velLimitValue->setStyleSheet("QLabel { color: #f57c00; font-weight: 600; }"); // 橙色：过低
            }
        } else {
            m_velLimitValue->setText("--");
            m_velLimitValue->setStyleSheet("");
        }
    }
    
    if (m_velRampRateValue) {
        if (status.velRampRate > 0.0f) {
            m_velRampRateValue->setText(QString("%1 转/秒²").arg(QString::number(status.velRampRate, 'f', 2)));
            
            // 根据加速度值设置颜色
            if (status.velRampRate > 2.0f) {
                m_velRampRateValue->setStyleSheet("QLabel { color: #f57c00; font-weight: 600; }"); // 橙色：过高
            } else {
                m_velRampRateValue->setStyleSheet("QLabel { color: #388e3c; font-weight: 600; }"); // 绿色：合理
            }
        } else {
            m_velRampRateValue->setText("--");
            m_velRampRateValue->setStyleSheet("");
        }
    }
    
    if (m_posGainValue) {
        if (status.posGain > 0.0f) {
            m_posGainValue->setText(QString::number(status.posGain, 'f', 1));
            
            // 根据位置增益设置颜色
            if (status.posGain > 20.0f) {
                m_posGainValue->setStyleSheet("QLabel { color: #f57c00; font-weight: 600; }"); // 橙色：过高
            } else {
                m_posGainValue->setStyleSheet("QLabel { color: #388e3c; font-weight: 600; }"); // 绿色：合理
            }
        } else {
            m_posGainValue->setText("--");
            m_posGainValue->setStyleSheet("");
        }
    }
}
```

## 颜色编码说明

### 速度限制 (vel_limit)
- **绿色** (0.5 - 2.0 转/秒): 合理范围，安全可控
- **橙色** (2.0 - 3.0 转/秒): 较高，需要注意
- **红色** (> 3.0 转/秒): 过高，难以控制
- **橙色** (< 0.5 转/秒): 过低，可能无法完成动作

### 加速度限制 (vel_ramp_rate)
- **绿色** (≤ 2.0 转/秒²): 合理范围
- **橙色** (> 2.0 转/秒²): 过高，启动太快

### 位置增益 (pos_gain)
- **绿色** (≤ 20.0): 合理范围
- **橙色** (> 20.0): 过高，可能导致震荡

## CAN 命令实现

需要实现 `Get_Controller_Config` (0x014) 命令来获取配置信息。

### 命令格式
- **命令 ID**: `(nodeId << 5) | 0x014`
- **DLC**: 0
- **响应**: 包含 vel_limit, vel_ramp_rate, pos_gain 等配置信息

### 响应解析
响应数据格式（根据 ODrive CAN 协议）：
- Bytes 0-3: vel_limit (float)
- Bytes 4-7: vel_ramp_rate (float)  
- 需要查看 ODrive 文档确认具体字节布局

## 实现步骤

1. 修改 `mainwindow.h` 添加新的标签成员变量
2. 修改 `odrivemotorcontroller.h` 的 `AxisStatus` 结构
3. 在 `odrivemotorcontroller.cpp` 中实现 `Get_Controller_Config` 命令
4. 在 `mainwindow.cpp` 的 `buildAdvancedContent` 中添加显示控件
5. 在 `updateStatusPanel` 中更新显示和颜色编码
6. 编译测试

## 测试验证

1. 连接 ODrive
2. 打开高级面板
3. 查看速度配置显示是否正确
4. 修改速度限制（通过 Set_Limits 命令）
5. 验证显示是否实时更新
6. 验证颜色编码是否正确（绿色/橙色/红色）

## 预期效果

用户可以在高级面板中实时看到：
```
速度限制: 3.0 转/秒 (红色，表示过高)
加速度限制: 0.5 转/秒² (绿色，表示合理)
位置增益: 8.0 (绿色，表示合理)
```

这样用户可以立即发现速度配置问题，避免电机运动过快或过慢。

## 注意事项

1. 需要定期发送 `Get_Controller_Config` 命令来更新配置信息
2. 可以在 `refreshTelemetry` 函数中添加配置查询
3. 颜色阈值可以根据实际使用情况调整
4. 需要处理配置信息不可用的情况（显示 "--"）
