# 节点0速度太快原因分析

## 问题描述

节点0（axis0）的电机运动速度太快，难以控制。

## 诊断结果

通过 `diagnose_speed_issue.py` 脚本诊断，发现以下问题：

### 当前配置

```
速度限制 (vel_limit): 3.0 转/秒 ⚠ 过高
加速度限制 (vel_ramp_rate): 0.5 转/秒² ✓ 合理
位置增益 (pos_gain): 8.0 ✓ 合理
速度增益 (vel_gain): 0.083
速度积分增益 (vel_integrator_gain): 0.333
```

### 根本原因

**速度限制设置为 3.0 转/秒，这是导致速度太快的主要原因。**

## 性能分析

以当前配置（3.0 转/秒）：
- 移动 1 转只需要 **0.3 秒**（太快）
- 移动 5 转只需要 **1.7 秒**
- 移动 10 转只需要 **3.3 秒**
- 加速到最大速度需要 **6.0 秒**

这个速度对于手动控制来说太快，难以精确控制和观察电机运动。

## 为什么会设置为 3.0 转/秒？

回顾调试历史：

1. **初始问题**：速度设置为 0.2 转/秒（太慢）
   - 移动 10 转需要 50 秒
   - 导致控制器超时失败（CONTROL_DEADLINE_MISSED）

2. **第一次调整**：提高到 1.0 转/秒
   - 使用 `set_balanced_speed.py` 设置
   - 这是一个合理的平衡值

3. **问题**：不知何时被改为 3.0 转/秒
   - 可能是通过 Qt 程序的 Set_Limits 命令修改的
   - 从日志中看到：`Set_Limits payload=00 00 80 40` 可能对应不同的速度值

## 解决方案

### 推荐配置（安全且可控）

```python
速度限制: 1.0 转/秒
加速度: 0.5 转/秒²
位置增益: 8.0
```

**性能表现：**
- 移动 1 转需要 1 秒（可控）
- 移动 5 转需要 5 秒
- 移动 10 转需要 10 秒
- 既安全又能完成动作

### 如果需要更慢

```python
速度限制: 0.5 转/秒
加速度: 0.3 转/秒²
位置增益: 5.0
```

**性能表现：**
- 移动 1 转需要 2 秒（很慢）
- 移动 5 转需要 10 秒
- 移动 10 转需要 20 秒
- 非常安全，适合精确控制

### 如果需要稍快

```python
速度限制: 2.0 转/秒
加速度: 1.0 转/秒²
位置增益: 10.0
```

**性能表现：**
- 移动 1 转需要 0.5 秒
- 移动 5 转需要 2.5 秒
- 移动 10 转需要 5 秒
- 较快但仍可控

## 如何修改配置

### 方法1：使用诊断脚本（推荐）

```bash
python diagnose_speed_issue.py
```

脚本会自动分析当前配置并提供修改选项。

### 方法2：使用 Python 直接修改

```python
import odrive
odrv0 = odrive.find_any()

# 设置推荐配置
odrv0.axis0.controller.config.vel_limit = 1.0
odrv0.axis0.controller.config.vel_ramp_rate = 0.5
odrv0.axis0.controller.config.pos_gain = 8.0

# 保存配置
odrv0.save_configuration()
```

### 方法3：使用现有脚本

```bash
# 设置平衡速度（1.0 转/秒）
python set_balanced_speed.py

# 或设置非常慢的速度（0.5 转/秒）
python make_very_slow.py

# 或设置较慢速度（2.0 转/秒）
python slow_down_motor.py
```

## Qt 程序中的注意事项

### 问题：Qt 程序可能会覆盖速度设置

从日志中看到，Qt 程序会发送 `Set_Limits` CAN 命令来设置速度限制：

```
[14:53:25] CANDLE TX Set_Limits id=0X00F dlc=8 payload=00 00 80 40 00 00 00 40
```

这个命令可能会覆盖 ODrive 中保存的速度配置。

### 解决方法

1. **检查 Qt 程序的速度设置**
   - 在 Qt 程序界面中查找速度限制设置
   - 确保设置为合理的值（如 1.0 转/秒）

2. **修改 Qt 程序代码**
   - 如果 Qt 程序硬编码了速度值，需要修改源代码
   - 查看 `mainwindow.cpp` 或 `odrivemotorcontroller.cpp` 中的速度设置

3. **禁用 Qt 程序的速度设置**
   - 如果可能，让 Qt 程序使用 ODrive 中保存的配置
   - 而不是每次都发送 Set_Limits 命令

## 总结

### 问题根源

**速度限制设置为 3.0 转/秒，导致电机运动太快。**

### 原因

1. 之前为了解决"速度太慢无法完成动作"的问题，提高了速度
2. 可能通过 Qt 程序或其他方式被设置为 3.0 转/秒
3. 这个值对于手动控制来说太快了

### 解决方案

1. 使用 `diagnose_speed_issue.py` 诊断并修改配置
2. 推荐设置为 1.0 转/秒（平衡值）
3. 检查 Qt 程序是否会覆盖速度设置
4. 根据实际需求调整速度（0.5-2.0 转/秒）

### 最佳实践

- **初次测试**：使用 0.5 转/秒（安全）
- **正常使用**：使用 1.0 转/秒（平衡）
- **快速移动**：使用 2.0 转/秒（较快但可控）
- **避免使用**：超过 3.0 转/秒（太快）
