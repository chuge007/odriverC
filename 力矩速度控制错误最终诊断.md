## 问题描述
用户反馈：使用力矩控制和速度控制电机时会发生错误

## 当前状态检查

电机错误: 0x00000000 (无错误)
速度限制: 3.0 turns/s

```

## 关键发现

### 1. 节点0和节点1都有错误，但错误类型不同

- **节点0**: DC_BUS_UNDER_VOLTAGE（欠压）
- **节点1**: WATCHDOG_TIMER_EXPIRED（看门狗超时）

### 2. 之前的 OVERSPEED 分析是误导

通过对比发现：
- 节点1的速度限制只有 3.0 turns/s（非常低）
- 节点1的位置增益高达 20.0（非常高）
- **但用户说节点1能正常工作！**

这说明问题不在速度限制或PID参数设置上。

### 3. 真正的问题：电源和通信问题

#### 节点0的欠压错误（0x00000002）
- 电源电压低于最小工作电压
- 可能原因：
  1. 电源供电不足
  2. 电源线接触不良
  3. 电机负载过大导致电压跌落
  4. 电源容量不够

#### 节点1的看门狗超时（0x00004000）
- ODrive 没有在规定时间内收到心跳信号
- 可能原因：
  1. CAN 通信中断
  2. 控制程序没有定期发送心跳
  3. CAN 总线负载过高

## 从日志分析的真相

查看之前的日志（10:54-10:58）：

```

当时的 CONTROLLER_FAILED 错误是由底层的欠压问题引起的：
1. 发送力矩控制命令
2. 电机尝试输出力矩
3. 电流增大导致电压跌落
4. 触发欠压保护
5. 控制器失败

## 解决方案

### 方案1：解决电源问题（优先）

```python
import odrive
odrv = odrive.find_any()

# 检查电源电压
print(f"母线电压: {odrv.vbus_voltage} V")

# ODrive 的工作电压范围通常是 12-56V
# 如果电压低于 12V，需要：
# 1. 检查电源输出电压
# 2. 检查电源线连接
# 3. 使用更大容量的电源（建议 24V 10A 以上）
```

**建议：**
- 使用 24V 或更高电压的电源
- 电源容量至少 10A（两个电机）
- 检查电源线是否松动
- 添加大容量电解电容（1000uF 50V）在电源输入端

### 方案2：降低电流限制（临时方案）

如果无法更换电源，可以降低电流限制来减少电压跌落：

```python
import odrive
odrv = odrive.find_any()

# 清除错误
odrv.axis0.error = 0
odrv.axis1.error = 0

# 降低电流限制
odrv.axis0.motor.config.current_lim = 10.0  # 从 20A 降到 10A
odrv.axis1.motor.config.current_lim = 10.0

# 保存配置
odrv.save_configuration()
odrv.reboot()
```

### 方案3：修复看门狗超时（节点1）

在 Qt 程序中添加定期心跳发送：

```cpp
// 在 mainwindow.cpp 中添加定时器
void MainWindow::setupHeartbeatTimer()
{
    QTimer *heartbeatTimer = new QTimer(this);
    connect(heartbeatTimer, &QTimer::timeout, this, [this]() {
        // 每 100ms 发送一次心跳
        for (int i = 0; i < m_boardRuntimes.size(); ++i) {
            ODriveMotorController *controller = m_boardRuntimes[i].controller;
            if (controller && controller->isConnected()) {
                // 发送心跳或状态查询
                controller->requestStatus(0);
                controller->requestStatus(1);
            }
        }
    });
    heartbeatTimer->start(100);  // 100ms 间隔
}
```

### 方案4：正确的控制流程

无论使用哪种控制模式，都要遵循正确的流程：

```python
import odrive
import time

odrv = odrive.find_any()

# 1. 检查电源电压
print(f"母线电压: {odrv.vbus_voltage} V")
if odrv.vbus_voltage < 12.0:
    print("警告：电压过低！")
    exit(1)

# 2. 清除错误
odrv.axis0.error = 0
odrv.axis0.controller.error = 0
odrv.axis0.motor.error = 0

# 3. 进入闭环
odrv.axis0.requested_state = 8
time.sleep(2)

# 4. 检查是否成功进入闭环
if odrv.axis0.current_state != 8:
    print(f"无法进入闭环，错误: 0x{odrv.axis0.error:08X}")
    exit(1)

# 5. 设置控制模式和发送命令
odrv.axis0.controller.config.control_mode = 2  # 速度控制
odrv.axis0.controller.input_vel = 1.0

# 或者力矩控制
# odrv.axis0.controller.config.control_mode = 1
# odrv.axis0.controller.input_torque = 0.5

# 6. 监控状态
time.sleep(2)
print(f"当前速度: {odrv.axis0.encoder.vel_estimate} turns/s")
print(f"母线电压: {odrv.vbus_voltage} V")
print(f"控制器错误: 0x{odrv.axis0.controller.error:08X}")
```

## 测试脚本

创建一个完整的测试脚本：

```python
import odrive
import time

def test_motor_control():
    print("连接 ODrive...")
    odrv = odrive.find_any()
    
    # 检查电源
    print(f"\n母线电压: {odrv.vbus_voltage:.2f} V")
    if odrv.vbus_voltage < 12.0:
        print("❌ 电压过低！请检查电源。")
        return False
    
    # 清除错误
    print("\n清除错误...")
    odrv.axis0.error = 0
    odrv.axis0.controller.error = 0
    odrv.axis0.motor.error = 0
    time.sleep(0.5)
    
    # 进入闭环
    print("进入闭环...")
    odrv.axis0.requested_state = 8
    time.sleep(2)
    
    if odrv.axis0.current_state != 8:
        print(f"❌ 无法进入闭环")
        print(f"轴错误: 0x{odrv.axis0.error:08X}")
        print(f"电机错误: 0x{odrv.axis0.motor.error:08X}")
        return False
    
    print("✓ 成功进入闭环")
    
    # 测试速度控制
    print("\n测试速度控制 (1.0 turns/s)...")
    odrv.axis0.controller.config.control_mode = 2
    odrv.axis0.controller.input_vel = 1.0
    time.sleep(2)
    
    vel = odrv.axis0.encoder.vel_estimate
    vbus = odrv.vbus_voltage
    ctrl_err = odrv.axis0.controller.error
    
    print(f"当前速度: {vel:.2f} turns/s")
    print(f"母线电压: {vbus:.2f} V")
    print(f"控制器错误: 0x{ctrl_err:08X}")
    
    if ctrl_err != 0:
        print("❌ 速度控制失败")
        return False
    
    # 停止
    odrv.axis0.controller.input_vel = 0
    time.sleep(1)
    
    # 测试力矩控制
    print("\n测试力矩控制 (0.3 Nm)...")
    odrv.axis0.controller.config.control_mode = 1
    odrv.axis0.controller.input_torque = 0.3
    time.sleep(2)
    
    vel = odrv.axis0.encoder.vel_estimate
    vbus = odrv.vbus_voltage
    ctrl_err = odrv.axis0.controller.error
    
    print(f"当前速度: {vel:.2f} turns/s")
    print(f"母线电压: {vbus:.2f} V")
    print(f"控制器错误: 0x{ctrl_err:08X}")
    
    if ctrl_err != 0:
        print("❌ 力矩控制失败")
        return False
    
    # 停止
    odrv.axis0.controller.input_torque = 0
    odrv.axis0.requested_state = 1
    
    print("\n✓ 所有测试通过！")
    return True

if __name__ == "__main__":
    test_motor_control()
```

## 总结

### 真正的问题
1. **节点0**: 电源欠压（DC_BUS_UNDER_VOLTAGE）
2. **节点1**: 看门狗超时（WATCHDOG_TIMER_EXPIRED）
3. **不是速度限制或PID参数的问题**

### 修复优先级
1. **高优先级**: 检查并升级电源（24V 10A+）
2. **高优先级**: 添加定期心跳发送
3. **中优先级**: 降低电流限制（临时方案）
4. **低优先级**: 优化PID参数（不是主要问题）

### 预期效果
修复电源问题后，力矩控制和速度控制应该能正常工作。
