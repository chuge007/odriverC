# 电机震动问题最终解决方案

## 诊断时间
2026/4/16 14:46 - 15:10

## 问题描述
- 节点0电机能转动，但有震动
- 节点1电机正常，无震动

---

## 🔍 诊断过程

### 第一步：使用 odrivetool 深度诊断
运行 `diagnose_vibration.py` 发现：

```
轴错误: 0x00004000 (CONTROLLER_FAILED)
Iq设定: -10.000 A
Iq测量: -7.647 A
Iq跟踪误差: 2.353 A
电流限制: 10.0 A
```

**问题1**：`CONTROLLER_FAILED` 错误
- 电流限制不足（10A）
- 电流跟踪误差过大

### 第二步：修复 CONTROLLER_FAILED
运行 `fix_controller_failed.py`：
- 清除错误
- 增加电流限制到 20A
- 结果：电机可以转动了 ✅

### 第三步：解决震动问题
虽然电机能转，但仍有震动。

**诊断结果**：
```
当前PID参数:
- 位置增益: 8.000 ✅
- 速度增益: 0.083333 ⚠️ (偏高)
- 速度积分增益: 0.320000 ⚠️ (偏高)
- 电流限制: 10.0 A ⚠️ (不足)
```

**问题2**：PID参数不合适
- 速度增益 (vel_gain) 过高 → 导致高频震动
- 速度积分增益 (vel_integrator_gain) 过高 → 导致震荡

---

## ✅ 最终解决方案

### 已应用的修复

运行 `fix_vibration_pid.py` 并应用以下参数：

```python
# 修复前
pos_gain = 8.000
vel_gain = 0.083333  ← 过高
vel_integrator_gain = 0.320000  ← 过高
current_lim = 10.0 A  ← 不足

# 修复后
pos_gain = 8.000  ← 保持不变
vel_gain = 0.050000  ← 降低 40%
vel_integrator_gain = 0.200000  ← 降低 37.5%
current_lim = 20.0 A  ← 增加 100%
```

### 参数已保存到 ODrive
配置已永久保存，重启后仍然有效。
