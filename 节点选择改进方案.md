# ODrive节点自动检测和选择改进方案

## 问题分析

当前问题：
1. 程序检测到多个ODrive节点（如Node 16和Node 1）
2. 用户不知道应该使用哪个节点
3. 需要手动在"高级"面板中修改节点ID
4. 轴状态显示不正确，因为可能在监控错误的节点

## 解决方案

### 1. 自动节点检测和显示

连接成功后，程序应该：
- 自动扫描并显示所有检测到的活跃节点
- 在主界面显示检测到的节点列表
- 让用户可以快速选择要控制的节点

### 2. 实现步骤

#### 步骤1：在主界面添加节点选择控件

在连接按钮旁边添加一个下拉框，显示检测到的节点：

```cpp
void MainWindow::toggleConnection()
{
    if (m_controller->isConnected()) {
        // ... 断开连接逻辑
        m_detectedNodesCombo->clear();
        m_detectedNodesCombo->setEnabled(false);
        m_rescanNodesButton->setEnabled(false);
        m_activeNodesLabel->setText("检测到的节点: --");
        return;
    }

    // ... 连接逻辑
    if (connected) {
        // 连接成功后，启动节点扫描
        m_rescanNodesButton->setEnabled(true);
        scanForActiveNodes();
    }
}
```

#### 步骤3：实现节点扫描功能

```cpp
void MainWindow::scanForActiveNodes()
{
    if (!m_controller->isConnected()) {
        return;
    }

    appendLog("正在扫描活跃的ODrive节点...");
    m_detectedNodesCombo->clear();
    m_detectedNodesCombo->setEnabled(false);
    // 探测所有节点
    m_controller->probeAllNodes();
    
    // 等待1.5秒收集响应
    QTimer::singleShot(1500, this, &MainWindow::updateDetectedNodesList);
}

void MainWindow::updateDetectedNodesList()
{
    if (!m_controller->isConnected()) {
        return;
    }

    // 获取最近有响应的节点
    const QList<quint8> activeNodes = m_controller->recentResponsiveNodeIds(2000);
    
    m_detectedNodesCombo->clear();
    if (activeNodes.isEmpty()) {
        m_activeNodesLabel->setText("检测到的节点: 无");
        appendLog("未检测到任何ODrive节点响应");
        return;
    }

    // 填充下拉框
    for (quint8 nodeId : activeNodes) {
        const QString nodeText = QString("Node %1").arg(nodeId);
        m_detectedNodesCombo->addItem(nodeText, nodeId);
    }
    
    m_detectedNodesCombo->setEnabled(true);
    // 更新标签
    QStringList nodeStrings;
    for (quint8 nodeId : activeNodes) {
        nodeStrings << QString::number(nodeId);
    }
    m_activeNodesLabel->setText(QString("检测到的节点: %1").arg(nodeStrings.join(", ")));
    appendLog(QString("检测到 %1 个活跃节点: %2")
              .arg(activeNodes.size())
              .arg(nodeStrings.join(", ")));
    // 如果当前配置的节点在列表中，自动选择它
    const int currentNodeIndex = m_detectedNodesCombo->findData(m_nodeIdSpin->value());
    if (currentNodeIndex >= 0) {
        m_detectedNodesCombo->setCurrentIndex(currentNodeIndex);
    } else if (!activeNodes.isEmpty()) {
        // 否则选择第一个检测到的节点
        m_detectedNodesCombo->setCurrentIndex(0);
        switchToDetectedNode(activeNodes.first());
    }
}
```

#### 步骤4：实现节点切换功能

```cpp
void MainWindow::switchToDetectedNode(quint8 nodeId)
{
    if (m_nodeIdSpin) {
        m_nodeIdSpin->setValue(nodeId);
    }
    
    m_controller->setDefaultNodeId(nodeId);
    updateTrackedNodes();
    m_controller->requestTrackedTelemetry();
    
    appendLog(QString("已切换到 Node %1").arg(nodeId));
    updateProgramStatus(QString("当前控制节点: %1").arg(nodeId));
}

// 连接下拉框的信号
connect(m_detectedNodesCombo, QOverload<int>::of(&QComboBox::currentIndexChanged),
        this, [this](int index) {
    if (index < 0 || !m_controller->isConnected()) {
        return;
    }
    const quint8 nodeId = m_detectedNodesCombo->currentData().toUInt();
    switchToDetectedNode(nodeId);
});

connect(m_rescanNodesButton, &QPushButton::clicked,
        this, &MainWindow::scanForActiveNodes);
```

```cpp
    const QDateTime lastHeartbeat = status.lastHeartbeat;
```

### 4. 用户体验改进

1. **连接时自动扫描**：连接成功后自动扫描节点
2. **可视化节点状态**：用不同颜色显示节点状态（在线/离线）
3. **快速切换**：通过下拉框快速切换控制的节点
4. **重新扫描**：提供"重新扫描"按钮，随时刷新节点列表
5. **智能默认选择**：如果配置的节点不在线，自动选择第一个在线的节点

## 实施优先级

1. **高优先级**：添加节点选择下拉框和自动扫描功能
2. **中优先级**：改进状态显示，显示当前控制的节点
3. **低优先级**：添加节点状态可视化（颜色指示）

## 预期效果

实施后：
- 用户连接后立即看到所有可用的ODrive节点
- 可以轻松选择要控制的节点
- 轴状态显示会正确反映所选节点的状态
- 不需要手动在"高级"面板中修改节点ID
