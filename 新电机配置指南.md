# ODrive 电机使用指南

## 🚀 快速开始

### 方案1：已校准的电机（推荐）

如果你的电机已经校准过，直接运行：

```bash
python auto_start_motor.py
```

这个脚本会自动启动电机进入闭环控制，无需任何交互。

### 方案2：新电机（需要校准）

如果是全新的电机，需要完整校准：

```bash
python standard_motor_config.py
```

### 这个脚本会自动完成：
1. ✅ 配置标准参数（保守稳定的设置）
2. ✅ 执行完整校准
3. ✅ 测试闭环控制
4. ✅ 保存配置到ODrive
5. ✅ 可选：自动重启ODrive

### 使用前确保：
- ⚠️ 电机可以自由转动（没有负载）
- ⚠️ 编码器正确连接
- ⚠️ 电源电压充足 (>11V)

---

## 📋 标准配置参数

脚本使用以下保守稳定的参数：

### 电机参数
- 极对数: 7
- 校准电流: 10A
- 电流限制: 10A

### 编码器参数
- CPR: 8192
- 模式: 增量式编码器

### 控制器参数（保守设置，避免错误）
- 速度限制: 10 圈/秒
- 速度增益: 0.02（降低以提高稳定性）
- 速度积分增益: 0.1（降低以避免CONTROL_DEADLINE_MISSED）
- 位置增益: 8.0

---

## 🔧 如果需要修改参数

编辑 `standard_motor_config.py` 文件顶部的 `STANDARD_CONFIG` 字典：

```python
STANDARD_CONFIG = {
    'pole_pairs': 7,  # 根据你的电机修改
    'cpr': 8192,      # 根据你的编码器修改
    'vel_limit': 10.0,  # 速度限制
    # ... 其他参数
}
```

## 📊 Qt程序使用

配置完成后，使用Qt程序控制电机：

1. 打开 `debug/odriver.exe`
2. 点击"connect"连接ODrive
3. 点击"高级"打开高级面板
4. 使用"诊断修复"按钮（绿色）快速修复问题
5. 使用方向按钮控制电机运动

## 💡 常见问题

### Q: 电机已经校准过，怎么快速启动？
A: 运行 `python auto_start_motor.py`，无需任何交互

### Q: 节点1无法进入闭环（错误0x00000028）？
A: 
1. 运行 `python fix_node1.py` 专门修复节点1
2. 或在Qt程序中点击"诊断修复"按钮
3. 检查编码器和电机连接

### Q: 校准失败怎么办？
A: 
1. 确保电机可以自由转动（无负载）
2. 检查编码器连接
3. 确认电源电压 >11V
4. 重新运行 `python standard_motor_config.py`

### Q: 电机速度太快或不稳定？
A: 
1. 使用 `auto_start_motor.py`，它使用保守参数
2. 或手动降低 `vel_limit` 和 `vel_integrator_gain`
