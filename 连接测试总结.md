**USB Pipe Error (Errno 32)**
- 发生位置：`ctrl_transfer` 调用时
- 具体命令：`GS_USB_BREQ_HOST_FORMAT`
- 错误信息：`USBError: [Errno 32] Pipe error`

## 问题分析

这个错误通常有以下几种可能：

### 1. 设备固件不兼容
candleLight/gs_usb 设备有多个版本的固件，某些固件可能不完全支持所有的控制命令。

**解决方案**：
- 更新设备固件到最新版本
- 或者使用 SLCAN 模式（如果设备支持）

### 2. 设备状态异常
设备可能处于错误状态，需要完全重置。

**解决方案**：
- 拔掉 USB 线，等待 10 秒
- 重新插入 USB 线
- 重启电脑

### 3. Windows USB 栈问题
Windows 的 USB 栈有时会出现状态不一致的问题。

**解决方案**：
- 在设备管理器中卸载设备
- 重新插入 USB 线让 Windows 重新识别

## 技术细节

### USB Pipe Error 的含义
- Errno 32 = EPIPE (Broken pipe)
- 在 USB 上下文中，表示端点停止响应（stalled）
- 通常是因为设备拒绝了控制传输请求

### gs_usb 协议
candleLight 使用 gs_usb 协议，需要以下控制传输：
1. `GS_USB_BREQ_HOST_FORMAT` - 设置主机格式
2. `GS_USB_BREQ_MODE` - 设置模式（RESET/START）
3. `GS_USB_BREQ_BITTIMING` - 设置波特率

如果第一步就失败，说明设备可能：
- 不支持这个命令
- 处于错误状态
- 固件版本不兼容
