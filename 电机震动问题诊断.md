## 电机震动的根本原因

### 🔴 主要问题

#### 1. **速度前馈值设置错误** ⚠️⚠️⚠️
```
Set_Input_Vel = 2.0圈/秒 + 前馈2.0
```
**问题**：速度前馈值应该是力矩前馈（单位：Nm），而不是速度值！
- 正确：`Set_Input_Vel(velocity=2.0, torque_ff=0.0)`
- 错误：`Set_Input_Vel(velocity=2.0, torque_ff=2.0)` ← 当前设置

**影响**：
- 2.0 Nm 的力矩前馈对于小电机来说可能过大
- 导致电机在速度控制时产生额外的力矩扰动
- 引起震动和不稳定

#### 2. **在空闲状态下发送速度命令**
```
[13:44:36] Set_Axis_State = 1 (空闲)
[13:44:37] Set_Input_Vel = 2.0圈/秒
```
**问题**：电机在空闲状态下不应该接收运动命令
- 空闲状态下电机没有激活控制器
- 发送命令会被忽略或导致异常行为

#### 3. **频繁切换控制模式**
在短时间内（13:44:18-13:44:23）切换了4种控制模式：
- 力矩控制 → 速度控制 → 位置控制 → 电压控制

**问题**：
- 每次切换模式都会重置控制器状态
- 可能导致控制器积分项累积
- 引起瞬态响应和震动

#### 4. **限制值设置不一致**
```
[13:44:17] Set_Limits (速度=2.0圈/秒, 电流=2.0A)   ← 太低
[13:44:25] Set_Limits (速度=10圈/秒, 电流=10A)    ← 正常
```
**问题**：
- 2.0A 的电流限制对于大多数电机来说太低
- 可能导致电机无法产生足够的力矩
- 引起控制不稳定和震动

### 🟡 次要问题

#### 5. **重复发送相同命令**
```
[13:44:33] Set_Input_Vel = 2.0圈/秒
[13:44:35] Set_Input_Vel = 2.0圈/秒  ← 2秒后重复
```
虽然不会直接导致震动，但表明控制逻辑可能有问题。

## 解决方案

### 立即修复

#### 1. **修正速度命令的前馈参数** 🔧
```cpp
// 错误的代码
setVelocity(nodeId, 2.0, 2.0);  // velocity=2.0, torque_ff=2.0 ❌

// 正确的代码
setVelocity(nodeId, 2.0, 0.0);  // velocity=2.0, torque_ff=0.0 ✅
```

#### 2. **确保在闭环状态下发送命令** 🔧
```cpp
// 先进入闭环
requestClosedLoop(nodeId);
QThread::msleep(500);  // 等待状态切换

// 再发送速度命令
setVelocity(nodeId, 2.0, 0.0);
```

#### 3. **使用合理的限制值** 🔧
```cpp
// 根据电机规格设置合理的限制
setLimits(nodeId, 10.0, 20.0);  // 速度=10圈/秒, 电流=20A
```

#### 4. **避免频繁切换控制模式** 🔧
```cpp
// 选择一种控制模式并保持
setControllerMode(nodeId, VelocityControl, Passthrough);

// 发送速度命令
setVelocity(nodeId, 2.0, 0.0);

// 不要立即切换到其他模式
```

### 建议的测试流程

#### 步骤1：清除错误并设置限制

## 总结

**电机震动的主要原因**：
1. ❌ 速度命令的力矩前馈值设置为 2.0 Nm（应该是 0.0）
2. ❌ 在空闲状态下发送速度命令
3. ❌ 频繁切换控制模式
4. ❌ 电流限制设置过低（2.0A）

**修复优先级**：
1. 🔴 **高优先级**：修正速度前馈参数（设为0.0）
2. 🔴 **高优先级**：确保在闭环状态下发送命令
3. 🟡 **中优先级**：使用合理的限制值（电流≥10A）
4. 🟡 **中优先级**：避免频繁切换控制模式

修复这些问题后，电机震动应该会消失。
