## 问题描述

## 根本原因分析

### 1. **电机未进入闭环状态就发送控制命令** ⚠️

**问题：** 代码在电机还未进入闭环状态（State=8）时，就发送了力矩或速度控制命令。

**证据：**
- 10:54:53 发送力矩控制命令
- 10:54:55 报错 CONTROLLER_FAILED
- 10:57:04-10:57:06 发送速度控制命令（连续两次）
- 10:58:46 发送力矩控制命令（连续两次）
- 10:58:48 才尝试进入闭环（Set_Axis_State = 8）
- 10:58:51 报错 CONTROLLER_FAILED

### 2. **控制模式切换顺序错误** ⚠️

**正确顺序应该是：**
1. 清除错误 (Clear_Errors)
2. 进入闭环 (Set_Axis_State = 8)
3. 等待电机进入闭环状态
4. 设置控制模式 (Set_Controller_Mode)
5. 发送控制命令 (Set_Input_Vel / Set_Input_Torque)

**实际顺序（错误）：**
1. 设置控制模式 (Set_Controller_Mode)
2. 发送控制命令 (Set_Input_Vel / Set_Input_Torque) ❌ 电机还在IDLE状态
3. 进入闭环 (Set_Axis_State = 8)
4. 报错 CONTROLLER_FAILED

### 3. **重复发送控制命令** ⚠️

日志显示多次重复发送相同的控制命令：
```
[10:58:46] Set_Controller_Mode + Set_Input_Torque
[10:58:46] Set_Controller_Mode + Set_Input_Torque (重复)
```

这可能导致控制器状态混乱。

## 解决方案

### 方案1：修改 `sendVelocityCommand()` 和 `sendTorqueCommand()`

在发送控制命令前，确保电机已进入闭环状态：

```cpp
void MainWindow::sendVelocityCommand()
{
    ODriveMotorController *controller = currentDebugController();
    if (!controller) {
        return;
    }
    
    const quint8 nodeId = currentDebugNodeId();
    const int boardIndex = currentDebugBoardIndex();
    
    // ✓ 1. 检查并确保电机在闭环状态
    const ODriveMotorController::AxisStatus &status = controller->status(nodeId);
    if (status.axisState != ODriveMotorController::ClosedLoopControl) {
        appendLog(QString("节点 %1 未在闭环状态，正在进入闭环...").arg(nodeId));
        if (!ensureAxisClosedLoop(boardIndex, nodeId)) {
            appendLog(QString("节点 %1 无法进入闭环状态").arg(nodeId));
            return;
        }
        // 等待电机进入闭环
        QTimer::singleShot(500, this, [this]() {
            sendVelocityCommand(); // 重试
        });
        return;
    }
    
    // ✓ 2. 设置速度限制
    const double velocityLimit = qMax(qAbs(m_velocitySpin->value()) * 1.5, 10.0);
    controller->setLimits(nodeId,
                          static_cast<float>(velocityLimit),
                          static_cast<float>(currentLimitAmps()));
    
    // ✓ 3. 设置控制模式
    const ODriveMotorController::InputMode inputMode =
            static_cast<ODriveMotorController::InputMode>(m_inputModeCombo->currentData().toInt());
    controller->setControllerMode(nodeId, ODriveMotorController::VelocityControl, inputMode);
    
    // ✓ 4. 发送速度命令
    controller->setVelocity(nodeId,
                            static_cast<float>(m_velocitySpin->value()),
                            static_cast<float>(m_velocityTorqueFfSpin->value()));
    
    appendLog(QString("速度控制命令已发送: %1 turns/s").arg(m_velocitySpin->value()));
}

void MainWindow::sendTorqueCommand()
{
    ODriveMotorController *controller = currentDebugController();
    if (!controller) {
        return;
    }
    
    const quint8 nodeId = currentDebugNodeId();
    const int boardIndex = currentDebugBoardIndex();
    
    // ✓ 1. 检查并确保电机在闭环状态
    const ODriveMotorController::AxisStatus &status = controller->status(nodeId);
    if (status.axisState != ODriveMotorController::ClosedLoopControl) {
        appendLog(QString("节点 %1 未在闭环状态，正在进入闭环...").arg(nodeId));
        if (!ensureAxisClosedLoop(boardIndex, nodeId)) {
            appendLog(QString("节点 %1 无法进入闭环状态").arg(nodeId));
            return;
        }
        // 等待电机进入闭环
        QTimer::singleShot(500, this, [this]() {
            sendTorqueCommand(); // 重试
        });
        return;
    }
    
    // ✓ 2. 设置电流限制
    controller->setLimits(nodeId,
                          20.0f, // 速度限制（力矩模式下不太重要）
                          static_cast<float>(currentLimitAmps()));
    
    // ✓ 3. 设置控制模式
    const ODriveMotorController::InputMode inputMode =
            static_cast<ODriveMotorController::InputMode>(m_inputModeCombo->currentData().toInt());
    controller->setControllerMode(nodeId, ODriveMotorController::TorqueControl, inputMode);
    
    // ✓ 4. 发送力矩命令
    controller->setTorque(nodeId, static_cast<float>(m_torqueSpin->value()));
    
    appendLog(QString("力矩控制命令已发送: %1 Nm").arg(m_torqueSpin->value()));
}
```

### 方案2：添加状态检查辅助函数

```cpp
// 在 mainwindow.h 中添加
private:
    bool isAxisInClosedLoop(int boardIndex, quint8 nodeId) const;
    bool waitForClosedLoop(int boardIndex, quint8 nodeId, int timeoutMs = 2000);

// 在 mainwindow.cpp 中实现
bool MainWindow::isAxisInClosedLoop(int boardIndex, quint8 nodeId) const
{
    if (boardIndex < 0 || boardIndex >= m_boardRuntimes.size()) {
        return false;
    }
    ODriveMotorController *controller = m_boardRuntimes[boardIndex].controller;
    if (!controller || !controller->isConnected()) {
        return false;
    }
    const ODriveMotorController::AxisStatus &status = controller->status(nodeId);
    return status.axisState == ODriveMotorController::ClosedLoopControl;
}

bool MainWindow::waitForClosedLoop(int boardIndex, quint8 nodeId, int timeoutMs)
{
    const qint64 deadline = QDateTime::currentMSecsSinceEpoch() + timeoutMs;
    while (QDateTime::currentMSecsSinceEpoch() < deadline) {
        if (isAxisInClosedLoop(boardIndex, nodeId)) {
            return true;
        }
        QThread::msleep(100);
        QCoreApplication::processEvents();
    }
    return false;
}
```

## 总结

### 问题根源
1. **时序错误**：在电机未进入闭环状态时发送控制命令
2. **缺少状态检查**：没有验证电机是否准备好接收控制命令
3. **重复命令**：可能导致控制器状态混乱

### 修复要点
1. ✅ 在发送控制命令前，确保电机已进入闭环状态
2. ✅ 添加状态检查和等待机制
3. ✅ 设置合适的速度和电流限制
4. ✅ 避免重复发送相同命令

### 建议测试步骤
1. 先点击"进入闭环"按钮
2. 等待电机状态显示为 State=8 (CLOSED_LOOP_CONTROL)
3. 再发送速度或力矩控制命令
4. 观察是否还会出现 CONTROLLER_FAILED 错误

### 额外建议
- 在UI上添加电机状态指示器，显示当前是否在闭环状态
- 禁用控制按钮，直到电机进入闭环状态
- 添加更详细的错误提示信息
