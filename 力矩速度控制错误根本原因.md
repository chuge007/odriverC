## 真正的问题根源

### 1. **OVERSPEED 错误 (0x00000001)** ⚠️⚠️⚠️

**这是主要问题！**

控制器错误显示 `OVERSPEED`，说明：
- 电机速度超过了配置的速度限制（12.0 turns/s）
- 当发送力矩或速度命令时，电机加速过快，超过了限制
- 这导致控制器立即报错并停止

**从日志分析：**
```
[10:54:53] Set_Input_Torque = 1.0 Nm
[10:54:55] 错误: CONTROLLER_FAILED
```

发送 1.0 Nm 的力矩后，电机可能瞬间加速超过 12 turns/s 的限制。

### 2. **CONTROL_DEADLINE_MISSED 错误 (0x00000010)** ⚠️

电机错误显示 `CONTROL_DEADLINE_MISSED`，说明：
- 控制循环没有在规定时间内完成
- 可能是因为 OVERSPEED 错误导致控制器停止响应
- 或者是 CAN 通信延迟导致

### 3. **速度限制设置过低** ⚠️

当前速度限制只有 **12.0 turns/s**，这对于：
- **力矩控制**：力矩可能导致快速加速，轻易超过限制
- **速度控制**：如果发送的速度命令接近或超过 12 turns/s，会立即报错

**从日志看到的速度命令：**
```
[10:57:04] Set_Input_Vel = 2.0 turns/s  ✓ 在限制内
[10:58:39] Set_Input_Vel = 2.0 turns/s  ✓ 在限制内
```

速度命令本身没问题，但问题在于：
1. 电机未在闭环状态就发送命令
2. 速度限制太低，无法应对力矩控制的加速

### 4. **PID 参数不合理** ⚠️

```
位置增益: 8.0  (太高！)
速度增益: 0.05
速度积分增益: 0.2
```

- **位置增益 8.0 太高**，会导致位置控制时响应过快
- 结合低速度限制，容易触发 OVERSPEED

## 解决方案

### 方案1：增加速度限制（推荐）

```python
import odrive
odrv = odrive.find_any()

# 清除错误
odrv.clear_errors()

# 增加速度限制到合理值
odrv.axis0.controller.config.vel_limit = 50.0  # 从 12 增加到 50 turns/s

# 保存配置
odrv.save_configuration()
odrv.reboot()
```

### 方案2：降低 PID 增益

```python
import odrive
odrv = odrive.find_any()

# 清除错误
odrv.clear_errors()

# 降低位置增益
odrv.axis0.controller.config.pos_gain = 1.0  # 从 8.0 降到 1.0

# 调整速度增益
odrv.axis0.controller.config.vel_gain = 0.02  # 从 0.05 降到 0.02
odrv.axis0.controller.config.vel_integrator_gain = 0.0  # 先关闭积分

# 保存配置
odrv.save_configuration()
odrv.reboot()
```

### 方案3：修改 Qt 程序中的限制设置

在 `mainwindow.cpp` 中修改速度限制：


### 方案4：在发送命令前设置更高的限制

```cpp
void MainWindow::sendVelocityCommand()
{
    ODriveMotorController *controller = currentDebugController();
    if (!controller) {
        return;
    }
    
    const quint8 nodeId = currentDebugNodeId();
    
    // ✓ 1. 确保电机在闭环状态
    ensureAxisClosedLoop(currentDebugBoardIndex(), nodeId);
    
    // ✓ 2. 设置足够高的速度限制（关键！）
    const double targetVel = qAbs(m_velocitySpin->value());
    const double safeLimit = qMax(targetVel * 5.0, 50.0);  // 至少是目标速度的5倍
    controller->setLimits(nodeId,
                          static_cast<float>(safeLimit),
                          static_cast<float>(currentLimitAmps()));
    
    // ✓ 3. 设置控制模式
    const ODriveMotorController::InputMode inputMode =
            static_cast<ODriveMotorController::InputMode>(m_inputModeCombo->currentData().toInt());
    controller->setControllerMode(nodeId, ODriveMotorController::VelocityControl, inputMode);
    
    // ✓ 4. 发送速度命令
    controller->setVelocity(nodeId,
                            static_cast<float>(m_velocitySpin->value()),
                            static_cast<float>(m_velocityTorqueFfSpin->value()));
}

void MainWindow::sendTorqueCommand()
{
    ODriveMotorController *controller = currentDebugController();
    if (!controller) {
        return;
    }
    
    const quint8 nodeId = currentDebugNodeId();
    
    // ✓ 1. 确保电机在闭环状态
    ensureAxisClosedLoop(currentDebugBoardIndex(), nodeId);
    
    // ✓ 2. 力矩控制需要更高的速度限制！
    controller->setLimits(nodeId,
                          100.0f,  // 设置很高的速度限制，允许力矩自由加速
                          static_cast<float>(currentLimitAmps()));
    
    // ✓ 3. 设置控制模式
    const ODriveMotorController::InputMode inputMode =
            static_cast<ODriveMotorController::InputMode>(m_inputModeCombo->currentData().toInt());
    controller->setControllerMode(nodeId, ODriveMotorController::TorqueControl, inputMode);
    
    // ✓ 4. 发送力矩命令
    controller->setTorque(nodeId, static_cast<float>(m_torqueSpin->value()));
}
```

## 总结

### 真正的问题
1. ⚠️ **速度限制太低（12 turns/s）** - 主要原因
2. ⚠️ **位置增益太高（8.0）** - 导致响应过快
3. ⚠️ **未在闭环状态发送命令** - 时序问题
4. ⚠️ **OVERSPEED 错误** - 速度超限导致控制器失败

### 修复优先级
1. **高优先级**：增加速度限制到 50+ turns/s
2. **高优先级**：降低位置增益到 1.0
3. **中优先级**：确保闭环状态再发送命令
4. **中优先级**：在 Qt 程序中设置更高的动态限制

### 预期效果
修复后，力矩控制和速度控制应该能正常工作，不再出现 CONTROLLER_FAILED 错误。
