## 问题根源

### 真正的原因：Qt 程序自动切换控制模式！

**不是电机或 ODrive 的问题，而是 Qt 程序的逻辑问题！**

力矩控制发送后，Qt 程序在 1-2 秒内自动切换到位置控制模式，导致：
1. 力矩控制命令生效
2. 电机开始转动
3. 1-2秒后程序自动切换到位置控制
4. 电机停止（因为位置控制会让电机保持在某个位置）

用户感觉就是"动一下就停止"！

## 代码问题定位

需要检查 `mainwindow.cpp` 中的以下部分：

### 可能的问题1：定时器自动切换模式

```cpp
// 可能存在某个定时器或信号槽，在发送力矩命令后自动切换模式
QTimer::singleShot(1000, this, [this]() {
    // 自动切换到位置控制？
    sendPositionCommand();
});
```

### 可能的问题2：UI 事件触发

```cpp
// 某个 UI 控件的状态变化触发了模式切换
void MainWindow::on_someWidget_valueChanged() {
    // 自动发送位置控制命令
}
```

### 可能的问题3：状态更新逻辑

```cpp
// 状态更新时自动切换到默认的位置控制模式
void MainWindow::updateStatus() {
    // ...
    if (某个条件) {
        setControlMode(PositionControl);
    }
}
```

## 解决方案

### 方案1：检查 mainwindow.cpp 中的自动切换逻辑

搜索以下关键词：
- `Set_Controller_Mode`
- `setControllerMode`
- `PositionControl`
- `Set_Input_Pos`
- `setPosition`

找出在发送力矩命令后，是什么触发了位置控制命令。

### 方案2：添加模式锁定机制

```cpp
// 在 mainwindow.h 中添加
private:
    bool m_controlModeLocked = false;
    ODriveMotorController::ControlMode m_currentControlMode;

// 在 mainwindow.cpp 中
void MainWindow::sendTorqueCommand()
{
    // ...
    m_controlModeLocked = true;
    m_currentControlMode = ODriveMotorController::TorqueControl;
    controller->setControllerMode(nodeId, ODriveMotorController::TorqueControl, inputMode);
    controller->setTorque(nodeId, torque);
}

void MainWindow::sendPositionCommand()
{
    // 如果模式被锁定，不允许切换
    if (m_controlModeLocked && m_currentControlMode != ODriveMotorController::PositionControl) {
        return;
    }
    // ...
}
```

### 方案3：禁用自动模式切换

找到触发自动切换的代码，添加条件判断：

```cpp
void MainWindow::someAutoSwitchFunction()
{
    // 只在 IDLE 状态或用户主动操作时才切换模式
    if (m_userInitiated) {
        setControlMode(...);
    }
}
```
