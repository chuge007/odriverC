## 发现的主要错误

### 1. USB Pipe Error (Errno 32) - 已修复 ✓

**错误位置**: `candle_bridge.py`

**错误描述**:
```
USBError: [Errno 32] Pipe error
File "D:/dev/odriveCar/debug/../candle_bridge.py", line 113, in open
    self.ctrl_out(GS_USB_BREQ_HOST_FORMAT, struct.pack("<I", 0x0000BEEF))
```

**根本原因**:
- USB设备初始化不稳定
- 控制传输命令没有重试机制
- 设备重置后等待时间不足
- 接口声明失败时没有重试

**修复方案**:
1. **改进设备初始化流程**:
   - 在重置前先尝试分离内核驱动
   - 增加设备重置后的等待时间（0.2秒 → 0.5秒）
   - 添加设备重置失败的容错处理

2. **添加重试机制**:
   - USB配置设置：最多重试3次
   - 接口声明：最多重试3次
   - 控制命令发送：最多重试3次
   - 控制传输：最多重试2次

3. **增加超时时间**:
   - 控制传输超时从1000ms增加到2000ms

4. **添加延迟**:
   - 每次重试之间添加0.1-0.3秒延迟
   - 确保USB设备有足够时间响应

### 2. Qt Widget Assertion Failure - 待观察

**错误位置**: `mainwindow.cpp`

**错误描述**:
```
[16:49:16.459] [FATAL] ASSERT failure in QWidget::mapTo(const QWidget *parent, const QPoint &pos): 
"parent must be in parent hierarchy", file kernel\qwidget.cpp, line 4253
```

**可能原因**:
- 某个Qt控件在父控件层次结构不正确时调用了mapTo()
- 可能是鼠标事件处理或工具提示显示时触发

**建议**:
- 这个错误较难重现，需要在实际使用中观察
- 如果再次出现，需要检查鼠标事件处理代码
- 可能与控件的显示/隐藏状态有关
