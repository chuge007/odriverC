# 电机不动问题诊断

## 当前代码逻辑（第2730-2760行）

按钮按下时的流程：
1. 获取所有已连接的板子索引 `connectedBoardIndices()`
2. 对每个板子，获取对应的节点ID（转向或驱动）
3. 调用 `commandAxisVelocity(boardIndex, nodeId, speed, turnAxis)`

## 可能的问题

### 问题1：没有连接的板子
如果 `connectedBoardIndices()` 返回空列表，按钮不会有任何动作。

**检查方法：**
在高级面板中查看：
- "板1 candle0" 的状态是否显示"已连接"？
- 点击"扫描"按钮后是否检测到节点16和节点1？

### 问题2：节点未进入闭环模式
`commandAxisVelocity` 函数会调用 `ensureAxisClosedLoop`，但如果节点有错误或未校准，可能无法进入闭环。

**检查方法：**
1. 在高级面板中，选择节点16或节点1
2. 点击"进入闭环"按钮
3. 查看"轴状态"是否变为 "CLOSED_LOOP_CONTROL (8)"
4. 查看"心跳错误"和"当前错误"是否为 0x00000000

### 问题3：速度太小
如果 `m_jogSpeedSpin->value()` 很小（比如0.1 mm/s），电机可能转得太慢看不出来。

**检查方法：**
将"点动速度"设置为较大值，比如 10.0 mm/s 或 50.0 mm/s

### 问题4：换算系数问题
`mmPerSecondToTurnsPerSecond` 使用 `boardScale` 进行换算。如果换算系数设置不对，可能导致速度命令错误。

**检查方法：**
在高级面板中查看：
- "转轴换算" 和 "驱轴换算" 的值
- 默认是 1.0 mm/圈，这意味着电机转1圈 = 移动1mm

## 推荐测试步骤

1. **连接并扫描节点**
   - 点击 connect 按钮
   - 打开高级面板
   - 点击"板1"的"扫描"按钮
   - 确认检测到节点

2. **手动进入闭环**
   - 在"节点切换"区域选择节点16
   - 点击"进入闭环"按钮
   - 等待轴状态变为 CLOSED_LOOP_CONTROL
   - 重复节点1

3. **测试速度控制**
   - 将"点动速度"设置为 10.0 mm/s
   - 按住"前进"按钮
   - 观察"接收信息"窗口是否有CAN消息
   - 观察电机是否转动

4. **如果还是不动**
   - 在高级面板的"速度控制"区域
   - 输入目标速度：1.0 圈/秒
   - 点击"发送速度"按钮
   - 看电机是否转动

## 可能需要的修改

如果上述测试发现问题，可能需要：
1. 修改换算系数（如果1圈不等于1mm）
2. 增加默认点动速度
3. 添加更多错误提示
4. 在按钮按下时自动进入闭环模式
