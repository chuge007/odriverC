# ODrive 调试完成总结

## 问题诊断和解决过程

### 问题1：CPR_POLEPAIRS_MISMATCH 错误

**症状：**
- Qt 程序中点击"整体校准"失败
- 无法进入闭环控制
- 编码器错误：CPR_POLEPAIRS_MISMATCH (0x00000002)
- 轴错误：ENCODER_FAILED (0x00000100)

**根本原因：**
编码器 CPR 配置错误
- 错误配置：CPR = 8192
- 正确配置：CPR = 16384
- 电机极对数：7

**解决方案：**
1. 使用 `check_status.py` 诊断出问题
2. 使用 `fix_cpr_mismatch.py` 修复 CPR 配置
3. 将 CPR 从 8192 改为 16384
4. 保存配置并重启

**结果：✓ 已解决**

---

### 问题2：电机运动太快

**症状：**
- 节点0的电机一下子运动太快
- 难以控制
- 不安全

**根本原因：**
速度和加速度限制设置过高

**解决方案：**
1. 使用 `slow_down_motor.py` 降低速度参数
2. 速度限制：从默认值降低到 2.0 转/秒
3. 加速度限制：设置为 1.0 转/秒²
4. 位置增益：从 20.0 降低到 10.0
5. 保存配置并重启

**结果：✓ 已解决**

## 使用说明

### 在 Qt 程序中操作

1. **连接到 ODrive**
   - 打开 Qt 程序
   - 选择板子和节点
   - 点击"连接"按钮

2. **进入闭环控制**
   - 点击"进入闭环"按钮
   - 系统应该成功进入闭环状态
   - 不会再出现 INVALID_STATE 错误

3. **控制电机**
   - 使用速度控制或位置控制
   - 电机现在运动速度慢（最大 2 转/秒）
   - 加速平滑（1 转/秒²）
   - 更加可控和安全

### 如果需要调整速度

**如果觉得太慢，可以提高速度：**
```python
import odrive
odrv0 = odrive.find_any()
odrv0.axis0.controller.config.vel_limit = 5.0  # 提高到 5 转/秒
odrv0.axis0.controller.config.vel_ramp_rate = 2.0  # 提高加速度
odrv0.save_configuration()
```

**如果觉得还是太快，可以进一步降低：**
```python
import odrive
odrv0 = odrive.find_any()
odrv0.axis0.controller.config.vel_limit = 1.0  # 降低到 1 转/秒
odrv0.axis0.controller.config.vel_ramp_rate = 0.5  # 降低加速度
odrv0.save_configuration()
```

## 技术细节

### CPR (Counts Per Revolution)
- CPR 是编码器每转一圈产生的脉冲数
- 你的编码器 CPR = 16384
- 必须与电机极对数匹配才能正常工作

### 速度和加速度控制
- 速度限制：控制电机最大转速
- 加速度限制：控制加速和减速的平滑度
- 位置增益：控制位置控制模式下的响应速度

### 电机参数
- 极对数: 7（已校准）
- 相电阻: 0.090265 Ω（已测量）
- 相电感: 0.000016535 H（已测量）
- 这些参数通过电机校准自动测量，无需手动修改
