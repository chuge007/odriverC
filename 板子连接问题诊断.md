# 板子连接问题诊断报告

## 问题现象
程序无法连接到 ODrive 板子，日志显示：
```
[11:38:19] [板1 candle0] [Errno 32] Pipe error
[11:38:19] [板1 candle0] candle process exited: 1
[11:38:19] [板1 candle0] connect failed
```

## 诊断结果

### ✅ 已确认正常的部分
1. **Python 环境**：Python 3.8.3 已安装
2. **USB 库**：libusb-package 1.0.26.3 和 pyusb 1.2.1 已安装
3. **设备检测**：candleLight 设备 (VID:0x1D50 PID:0x606F) 已被检测到
4. **candle_bridge.py 脚本**：脚本本身没有语法错误

### ❌ 问题根源
设备访问时出现 **"Error Accessing String"** 错误，这表明：
- Windows 系统检测到了设备
- 但是 **无法正确访问设备** (权限/驱动问题)
- 设备可能被其他程序占用，或者缺少 WinUSB 驱动

## 解决方案

### 方案 1：安装 WinUSB 驱动（推荐）

candleLight 设备在 Windows 上需要 WinUSB 驱动才能被 libusb 访问。

**步骤：**

1. **下载 Zadig 工具**
   - 访问：https://zadig.akeo.ie/
   - 下载最新版本的 Zadig

2. **安装 WinUSB 驱动**
   - 运行 Zadig
   - 点击菜单 `Options` → 勾选 `List All Devices`
   - 在下拉列表中找到 `gs_usb` 或 `candleLight` 设备
   - 在驱动选择框中选择 **WinUSB**
   - 点击 `Replace Driver` 或 `Install Driver`
   - 等待安装完成

3. **重新连接设备**
   - 拔掉 USB 线
   - 等待 5 秒
   - 重新插入 USB 线

4. **测试连接**
   - 运行程序
   - 点击 Connect 按钮

### 方案 2：检查设备占用

如果其他程序正在使用该设备，需要先关闭：

```powershell
# 检查是否有其他程序在使用 USB 设备
# 关闭可能占用设备的程序，例如：
# - CANgaroo
# - 其他 CAN 工具
# - 之前运行的 odriver.exe 实例
```

**步骤：**
1. 打开任务管理器 (Ctrl+Shift+Esc)
2. 查找并结束以下进程：
   - `odriver.exe`
   - `python.exe` (如果有多个)
   - 其他 CAN 相关工具
3. 重新启动程序

### 方案 3：使用管理员权限运行

有时候需要管理员权限才能访问 USB 设备：

1. 右键点击 `debug\odriver.exe`
2. 选择 **"以管理员身份运行"**
3. 尝试连接

### 方案 4：切换到 SLCAN 模式（备选方案）

如果 candleLight 模式一直有问题，可以尝试使用 SLCAN 模式：

1. 将 CAN 适配器设置为 SLCAN 模式（如果支持）
2. 在程序中：
   - 插件选择：`slcan` 或 `auto`
   - 接口选择：对应的 COM 口（如 `COM3`）
3. 点击 Connect

## 验证步骤

安装驱动后，运行以下命令验证：

```powershell
python -c "import usb.core; import libusb_package; import usb.backend.libusb1; backend = usb.backend.libusb1.get_backend(find_library=libusb_package.find_library); dev = usb.core.find(idVendor=0x1D50, idProduct=0x606F, backend=backend); print('Device found!' if dev else 'Device not found'); dev.set_configuration() if dev else None; print('Configuration set successfully!')"
```

如果看到：
```
Device found!
Configuration set successfully!
```
说明驱动安装成功！

## 常见问题

### Q1: Zadig 找不到设备？
**A:** 确保勾选了 `Options` → `List All Devices`

### Q2: 安装驱动后还是不行？
**A:** 
1. 重启电脑
2. 尝试不同的 USB 口
3. 检查 USB 线是否正常

### Q3: 提示 "Access Denied"？
**A:** 以管理员身份运行程序

### Q4: 设备在设备管理器中显示黄色感叹号？
**A:** 说明驱动有问题，需要重新安装 WinUSB 驱动

## 技术细节

- **设备信息**：VID=0x1D50, PID=0x606F (candleLight/gs_usb)
- **错误类型**：Errno 32 (Broken pipe) - 通常是设备访问权限问题
- **Python 版本**：3.8.3
- **USB 库版本**：libusb-package 1.0.26.3, pyusb 1.2.1
