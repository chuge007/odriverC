## 🔍 震动的可能原因

### 1. 如果是持续震动/抖动
**可能原因**：
- PID参数不合适（增益太高）
- 编码器噪声或连接问题
- 电机负载过大
- 电源电压不稳定

**检查方法**：
```python
# 使用 Python 脚本检查状态
import odrive
odrv0 = odrive.find_any()

# 检查编码器
print(f"编码器位置: {odrv0.axis0.encoder.pos_estimate}")
print(f"编码器速度: {odrv0.axis0.encoder.vel_estimate}")
print(f"编码器CPR: {odrv0.axis0.encoder.config.cpr}")

# 检查PID参数
print(f"位置增益: {odrv0.axis0.controller.config.pos_gain}")
print(f"速度增益: {odrv0.axis0.controller.config.vel_gain}")
print(f"速度积分增益: {odrv0.axis0.controller.config.vel_integrator_gain}")

# 检查电流
print(f"Iq设定: {odrv0.axis0.motor.current_control.Iq_setpoint}")
print(f"Iq测量: {odrv0.axis0.motor.current_control.Iq_measured}")
```

### 2. 如果是校准时的震动
**这是正常的！**
- 全校准会让电机旋转多圈
- 电机校准会测量相电阻和电感
- 编码器校准会寻找索引脉冲

### 3. 如果是切换模式时的瞬间震动
**原因**：控制器状态重置
**解决**：避免频繁切换模式

## ✅ 建议的稳定测试流程

### 步骤1：确保电机已校准
```
如果电机从未校准过：
1. Set_Axis_State = 3 (全校准)
2. 等待校准完成（可能需要10-30秒）
3. 检查 procedure_result 是否为 0（成功）

如果电机已校准：
跳过此步骤
```

### 步骤2：进入闭环并保持
```
1. Clear_Errors
2. Set_Limits (速度=10圈/秒, 电流=20A)
3. Set_Axis_State = 8 (闭环)
4. 等待2秒让状态稳定
```

### 步骤3：设置速度控制模式（只设置一次）
```
1. Set_Controller_Mode = 速度控制 + 透传模式
2. 不要再切换模式！
```

### 步骤4：发送速度命令并观察
```
1. Set_Input_Vel = 0.5圈/秒 + 0.0前馈
2. 观察5秒
3. 如果稳定，逐步增加：1.0 → 2.0 → 5.0
```

### 步骤5：停止
```
1. Set_Input_Vel = 0.0圈/秒 + 0.0前馈
2. 等待电机停止
3. Set_Axis_State = 1 (空闲)
```
