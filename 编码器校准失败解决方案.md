# 编码器校准失败问题诊断与解决

## 问题描述
在执行"整体校准"时，日志显示编码器校准失败，导致无法进入闭环状态。

## 常见原因分析

### 1. 编码器未连接或连接不良
- **症状**：校准时编码器无响应
- **检查**：
  - 编码器线缆是否松动
  - A、B、Z 相信号线是否正确连接
  - 电源线（5V/3.3V）是否连接

### 2. 编码器配置错误
- **症状**：编码器类型或参数设置不正确
- **常见错误**：
  - CPR（每转脉冲数）设置错误
  - 编码器模式设置错误
  - 带宽设置过低

### 3. 电机未正确配置
- **症状**：电机参数不匹配
- **检查**：
  - 极对数设置
  - 电阻和电感值
  - 电流限制

### 4. 电源电压不足
- **症状**：校准过程中电压下降
- **检查**：
  - 母线电压是否在正常范围（通常 12-48V）
  - 电源功率是否足够

## 解决方案

### 方案1：使用 odrivetool 手动配置编码器

```python
# 连接到 ODrive
import odrive
odrv0 = odrive.find_any()

# 配置编码器（以增量式编码器为例）
odrv0.axis0.encoder.config.mode = 0  # ENCODER_MODE_INCREMENTAL
odrv0.axis0.encoder.config.cpr = 8192  # 根据实际编码器设置，常见值：2048, 4096, 8192

# 配置编码器带宽（重要！）
odrv0.axis0.encoder.config.bandwidth = 1000  # Hz，可以尝试 100-3000

# 保存配置
odrv0.save_configuration()
odrv0.reboot()

# 重新连接后测试
odrv0 = odrive.find_any()
odrv0.axis0.requested_state = 7  # AXIS_STATE_ENCODER_INDEX_SEARCH
# 等待完成，观察 axis0.current_state 是否回到 1 (IDLE)

# 如果成功，再进行电机校准
odrv0.axis0.requested_state = 4  # AXIS_STATE_MOTOR_CALIBRATION
# 等待完成

# 最后进行完整校准
odrv0.axis0.requested_state = 3  # AXIS_STATE_FULL_CALIBRATION_SEQUENCE
```

### 方案2：跳过编码器索引搜索（临时方案）

如果编码器没有 Z 相（索引信号），可以跳过索引搜索：

```python
# 禁用索引搜索
odrv0.axis0.encoder.config.use_index = False

# 设置为在启动时不搜索索引
odrv0.axis0.config.startup_encoder_index_search = False

# 保存并重启
odrv0.save_configuration()
odrv0.reboot()
```

### 方案3：使用无传感器模式（最后手段）

如果编码器确实有问题，可以临时使用无传感器模式：

```python
# 配置为无传感器模式
odrv0.axis0.controller.config.control_mode = 2  # CONTROL_MODE_VELOCITY_CONTROL
odrv0.axis0.controller.config.input_mode = 1    # INPUT_MODE_VEL_RAMP

# 启用无传感器估计器
odrv0.axis0.sensorless_estimator.config.pm_flux_linkage = 5.51328895422 / (7 * 270)  # 根据电机参数计算

# 保存配置
odrv0.save_configuration()
odrv0.reboot()
```

### 方案4：检查和修复编码器配置（推荐）

#### 步骤1：检查当前配置
```python
import odrive
odrv0 = odrive.find_any()

# 查看编码器配置
print("编码器模式:", odrv0.axis0.encoder.config.mode)
print("CPR:", odrv0.axis0.encoder.config.cpr)
print("带宽:", odrv0.axis0.encoder.config.bandwidth)
print("使用索引:", odrv0.axis0.encoder.config.use_index)

# 查看编码器错误
print("编码器错误:", hex(odrv0.axis0.encoder.error))

# 查看编码器状态
print("编码器就绪:", odrv0.axis0.encoder.is_ready)
print("编码器计数:", odrv0.axis0.encoder.shadow_count)
```

#### 步骤2：根据编码器类型配置

**增量式编码器（最常见）：**
```python
odrv0.axis0.encoder.config.mode = 0  # INCREMENTAL
odrv0.axis0.encoder.config.cpr = 8192  # 根据实际编码器
odrv0.axis0.encoder.config.bandwidth = 1000
```

**霍尔传感器：**
```python
odrv0.axis0.encoder.config.mode = 1  # HALL
odrv0.axis0.encoder.config.cpr = 6 * odrv0.axis0.motor.config.pole_pairs
```

**SPI 绝对式编码器：**
```python
odrv0.axis0.encoder.config.mode = 257  # ABS_SPI
odrv0.axis0.encoder.config.abs_spi_cs_gpio_pin = 1
```

#### 步骤3：执行校准序列
```python
# 清除错误
odrv0.clear_errors()

# 先进行电机校准（不需要编码器）
odrv0.axis0.requested_state = 4  # MOTOR_CALIBRATION
import time
time.sleep(10)  # 等待完成

# 检查结果
if odrv0.axis0.motor.error != 0:
    print("电机校准失败:", hex(odrv0.axis0.motor.error))
else:
    print("电机校准成功")
    
    # 再进行编码器校准
    odrv0.axis0.requested_state = 7  # ENCODER_INDEX_SEARCH
    time.sleep(10)
    
    if odrv0.axis0.encoder.error != 0:
        print("编码器校准失败:", hex(odrv0.axis0.encoder.error))
    else:
        print("编码器校准成功")
```

## 常见编码器错误代码

| 错误代码 | 含义 | 解决方法 |
|---------|------|---------|
| 0x01 | UNSTABLE_GAIN | 降低编码器带宽或检查连接 |
| 0x02 | CPR_POLEPAIRS_MISMATCH | 检查 CPR 和极对数设置 |
| 0x04 | NO_RESPONSE | 编码器无响应，检查连接 |
| 0x08 | UNSUPPORTED_ENCODER_MODE | 编码器模式不支持 |
| 0x10 | ILLEGAL_HALL_STATE | 霍尔传感器状态非法 |
| 0x20 | INDEX_NOT_FOUND_YET | 未找到索引信号 |


## 快速修复脚本

创建一个 Python 脚本来自动配置和测试：

```python
#!/usr/bin/env python3
"""
ODrive 编码器快速配置脚本
"""
import odrive
import time

def configure_encoder(axis, cpr=8192):
    """配置增量式编码器"""
    print("配置编码器...")
    axis.encoder.config.mode = 0  # INCREMENTAL
    axis.encoder.config.cpr = cpr
    axis.encoder.config.bandwidth = 1000
    axis.encoder.config.use_index = True
    print(f"编码器配置完成: CPR={cpr}, 带宽=1000Hz")

def test_encoder_connection(axis):
    """测试编码器连接"""
    print("\n测试编码器连接...")
    count1 = axis.encoder.shadow_count
    print(f"当前计数: {count1}")
    input("请手动转动电机轴，然后按回车...")
    count2 = axis.encoder.shadow_count
    print(f"新计数: {count2}")
    
    if abs(count2 - count1) > 10:
        print("✓ 编码器连接正常")
        return True
    else:
        print("✗ 编码器可能未连接或配置错误")
        return False

def calibrate_motor(axis):
    """校准电机"""
    print("\n开始电机校准...")
    axis.requested_state = 4  # MOTOR_CALIBRATION
    
    while axis.current_state != 1:  # 等待回到 IDLE
        time.sleep(0.5)
        print(".", end="", flush=True)
    
    print()
    if axis.motor.error != 0:
        print(f"✗ 电机校准失败: {hex(axis.motor.error)}")
        return False
    else:
        print("✓ 电机校准成功")
        return True

def calibrate_encoder(axis):
    """校准编码器"""
    print("\n开始编码器校准...")
    axis.requested_state = 7  # ENCODER_INDEX_SEARCH
    
    while axis.current_state != 1:
        time.sleep(0.5)
        print(".", end="", flush=True)
    
    print()
    if axis.encoder.error != 0:
        print(f"✗ 编码器校准失败: {hex(axis.encoder.error)}")
        return False
    else:
        print("✓ 编码器校准成功")
        return True

def main():
    print("正在连接 ODrive...")
    odrv0 = odrive.find_any()
    print(f"已连接: {odrv0.serial_number}")
    
    # 清除错误
    odrv0.clear_errors()
    
    # 配置编码器
    cpr = int(input("请输入编码器 CPR (默认 8192): ") or "8192")
    configure_encoder(odrv0.axis0, cpr)
    
    # 测试连接
    if not test_encoder_connection(odrv0.axis0):
        print("\n请检查编码器连接后重试")
        return
    
    # 校准电机
    if not calibrate_motor(odrv0.axis0):
        print("\n电机校准失败，请检查电机连接和配置")
        return
    
    # 校准编码器
    if not calibrate_encoder(odrv0.axis0):
        print("\n编码器校准失败")
        print("尝试禁用索引搜索...")
        odrv0.axis0.encoder.config.use_index = False
        odrv0.save_configuration()
        odrv0.reboot()
        time.sleep(3)
        odrv0 = odrive.find_any()
        
        if calibrate_motor(odrv0.axis0):
            print("✓ 无索引模式下校准成功")
        else:
            print("✗ 校准仍然失败，请检查硬件")
            return
    
    # 保存配置
    print("\n保存配置...")
    odrv0.save_configuration()
    print("✓ 配置已保存")
    
    print("\n校准完成！现在可以尝试进入闭环控制。")

if __name__ == "__main__":
    main()
```

## 总结

编码器校准失败的解决步骤：

1. **检查硬件连接** - 确保编码器线缆连接牢固
2. **验证编码器配置** - 使用 odrivetool 检查 CPR 和模式设置
3. **测试编码器响应** - 手动转动电机，观察计数变化
4. **分步校准** - 先校准电机，再校准编码器
5. **调整参数** - 如果失败，尝试调整带宽或禁用索引搜索
6. **使用脚本** - 运行上面的快速配置脚本自动化流程

如果所有方法都失败，可能是编码器硬件故障，需要更换编码器或使用无传感器模式。
